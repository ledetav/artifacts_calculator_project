# Genshin Impact AI Build Optimizer

Проект представляет собой мобильное приложение для Android, предназначенное для помощи игрокам в управлении артефактами, оружием и персонажами из игры Genshin Impact. Ключевая особенность — использование **искусственного интеллекта (Gemini API)** для автоматического создания оптимальных билдов и командных сборок на основе имеющихся у пользователя предметов и персонажей.

## Технологический стек

*   **Фреймворк: Flutter.**
    *   **Почему Flutter?** Позволяет быстро создавать производительные и визуально привлекательные приложения для Android из единой кодовой базы. Идеально подходит для быстрой итеративной разработки.
    *   **Язык:** Dart.

*   **Распознавание текста (OCR): Google ML Kit (Text Recognition).**
    *   **Почему ML Kit?** Мощный и простой в интеграции инструмент для распознавания характеристик артефактов со скриншотов прямо на устройстве, поддерживает русский и английский языки.

*   **Искусственный интеллект: Google Gemini API.**
    *   **Почему Gemini?** Для создания гибких и контекстно-зависимых рекомендаций по сборкам. Gemini будет анализировать весь пул артефактов, оружия и персонажей пользователя, чтобы предложить наилучшие комбинации, адаптируясь под конкретные запросы (например, "собери мне этого персонажа в саппорта под реакцию 'пар'"). Это избавляет от необходимости жестко кодировать всю игровую логику.
    *   **Интеграция:** Через `http` запросы к API.

*   **База данных: SQLite.**
    *   **Почему SQLite?** Надежное и быстрое локальное хранилище для данных пользователя (артефакты, оружие, персонажи).

## План разработки (Agile)

Разработка будет вестись короткими итерациями (спринтами) с фокусом на поэтапной реализации функционала.

### **Спринт 1: Основа и ручной ввод (Завершение: 22 сентября 2025)**

**Цель:** Создать каркас приложения и реализовать базовую функцию ручного ввода артефактов.

*   [ ] Настроить рабочее окружение для Flutter.
*   [ ] Создать базовую структуру проекта: экраны, модели данных.
*   [ ] Реализовать UI-форму для ручного ввода одного артефакта (сет, часть сета, уровень, основной стат, подстаты).
*   [ ] Реализовать временное хранение добавленных артефактов в списке.
*   [ ] Создать простой экран для отображения списка добавленных артефактов.

### **Спринт 2: Расширение базы данных (Завершение: 6 октября 2025)**

**Цель:** Добавить возможность вводить оружие и персонажей, настроить локальную базу данных.

*   [ ] Спроектировать и создать таблицы в SQLite для артефактов, оружия и персонажей.
*   [ ] Перенести логику сохранения артефактов в базу данных SQLite.
*   [ ] Создать UI и логику для добавления оружия из предустановленного списка.
*   [ ] Создать UI и логику для добавления персонажей из предустановленного списка.
*   [ ] Реализовать экраны для просмотра списков имеющегося оружия и персонажей.

### **Спринт 3: OCR — Распознавание со скриншотов (Завершение: 20 октября 2025)**

**Цель:** Интегрировать систему распознавания текста для автоматического добавления артефактов.

*   [ ] Интегрировать Google ML Kit в проект.
*   [ ] Реализовать функцию выбора одного или нескольких скриншотов из галереи.
*   [ ] Разработать алгоритм для парсинга (анализа и структурирования) распознанного текста для извлечения характеристик артефакта.
*   [ ] Создать UI, где пользователь может проверить и при необходимости скорректировать распознанные данные перед сохранением в базу данных.

### **Спринт 4: Интеграция Gemini API и базовые билды (Завершение: 3 ноября 2025)**

**Цель:** Реализовать вызовы Gemini API для создания "дефолтных" билдов.

*   [ ] Настроить сетевой слой в приложении (добавить пакет `http`).
*   [ ] Интегрировать Gemini API. Реализовать безопасное управление API-ключом (например, через ввод пользователем).
*   [ ] Разработать функцию, которая собирает все имеющиеся у пользователя артефакты и оружие в структурированный формат (JSON/текст) для отправки в API.
*   [ ] **Prompt Engineering:** Сконструировать первый базовый запрос к Gemini: "Вот персонаж (уровень, таланты, созвездия). Вот полный список доступных артефактов. Подбери лучший набор из 5 артефактов для максимального урона, объясни выбор."
*   [ ] Реализовать парсинг ответа от Gemini, чтобы выделить рекомендованные артефакты и отобразить их в UI.

### **Спринт 5: Продвинутые билды и кастомизация через ИИ (Завершение: 17 ноября 2025)**

**Цель:** Расширить возможности ИИ, позволяя пользователю задавать конкретную роль для персонажа.

*   [ ] Добавить в UI "Сборщика билдов" выбор роли (физ. ДПС, саб-ДПС, саппорт и т.д.).
*   [ ] **Prompt Engineering:** Модифицировать промпт для Gemini, включив в него выбранную пользователем роль и список доступного оружия.
*   [ ] Реализовать обработку ответа от ИИ на вопрос: "Возможно ли собрать этого персонажа в такой роли с текущими предметами?". Отобразить объяснение от ИИ пользователю.

### **Спринт 6: Командная синергия с помощью ИИ (Завершение: 1 декабря 2025)**

**Цель:** Использовать Gemini для рекомендаций по составу отряда.

*   [ ] Разработать функцию для сбора данных о всех доступных персонажах пользователя.
*   [ ] **Prompt Engineering:** Сконструировать новый промпт для Gemini: "Вот основной персонаж и его билд. Вот список других моих персонажей. Посоветуй лучшую команду из 3-х компаньонов для синергии (учитывая элементальные реакции и роли) и объясни свой выбор."
*   [ ] Реализовать UI для отображения рекомендуемой команды и объяснения от ИИ.

### **Спринт 7: UX/UI и тестирование (Завершение: 15 декабря 2025)**

**Цель:** "Отполировать" приложение, исправить ошибки и улучшить пользовательский опыт.

*   [ ] Провести полное тестирование всех функций, включая API-запросы.
*   [ ] Оптимизировать производительность и обработку состояний (загрузка, ошибка) при обращении к Gemini.
*   [ ] Собрать обратную связь (если возможно) и внести корректировки в интерфейс.
*   [ ] Добавить пользовательские подсказки и инструкции.
*   [ ] Подготовить приложение к возможной публикации.
